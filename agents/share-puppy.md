# Agent: share-puppy
**Category:** Dev Agent  
**Type:** Code Puppy Sub-Agent  
**Display Name:** Share Puppy 📤

---

## What It Is
Publishes flat HTML pages, reports, and dashboards to `puppy.walmart.com/sharing`
so other associates can view them via a direct link — no server required on your end.

## Role in This Project
SiteOwlQA generates several HTML artifacts. When any of them need to be shared
with stakeholders outside the local machine, use share-puppy to publish them.

## Artifacts That Can Be Shared
| Artifact | Source | Description |
|---|---|---|
| Executive Dashboard | `output/executive_dashboard.html` | Main live dashboard — vendor performance, pipeline health |
| Orchestration Map | `orchestration_map.html` | Pipeline node map — 59 nodes across 8 drill-down realms |
| Weekly Highlights | generated by `weekly_highlights.py` | LLM-generated executive operations report |
| Memory Token Report | `docs/memory_token_optimization_report.md` | Converted to HTML if needed |

## How to Invoke
```
/agent share-puppy
```
Then provide the path to the HTML file you want to publish.

## When to Use It
- Sharing the weekly executive report with leadership
- Giving stakeholders a read-only view of the dashboard without exposing the server
- Sharing the orchestration map for architecture reviews
- Sending a one-off audit report (e.g., `memory_token_optimization_report.md`) to the team

## How the App Serves Locally (Not via share-puppy)
The dashboard is also served locally via nginx on port 8765 (Docker) or directly
via `served_dashboard/` (non-Docker). See `INFRA.md` for setup.
share-puppy is for external/stakeholder access only — local serving is separate.
