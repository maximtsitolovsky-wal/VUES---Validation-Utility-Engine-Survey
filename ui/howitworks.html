What is VUES?

🎯 The Mission

VUES (Vendor Upload Evaluation System) is an automated quality assurance platform for the SiteOwl Survey Program. It grades vendor-submitted survey data against reference baselines, tracks Scout site completion, and provides executive visibility into program health.

The system was built to eliminate manual grading, reduce human error, and provide real-time insights into vendor performance across hundreds of retail locations.

💡 The Problem It Solves
Before VUES, survey submissions were manually reviewed — a process that was slow, inconsistent, and couldn't scale. VUES automates this entirely, grading submissions in seconds with 100% deterministic accuracy.
How Does It Work?

⚙️ The Pipeline

VUES runs as a continuous background service that polls Airtable for new submissions, grades them automatically, and updates results in real-time.

Grading Logic: Each submission is compared row-by-row against the SQL Server reference database. If ≥95% of rows match correctly, the submission passes. Below 95% = FAIL with detailed mismatch report.

🏗️ Architecture

VUES is built as a single-process Python application with no external dependencies like Docker, message queues, or cloud services. Everything runs locally on a Windows machine.

Python 3.11+ Airtable API SQL Server FastAPI Chart.js Tailwind CSS Windows Task Scheduler

Key Design Principles:

  • Deterministic Grading: No AI/LLM in the grading path — results are 100% reproducible
  • Polling over Webhooks: Airtable doesn't support webhooks, so we poll reliably
  • Single Process: No Docker, no queues, no complexity — just Python
  • Local First: All data stays on-premises, nothing sent to external services
Use Cases

👥 Who Uses VUES?

  • Survey Vendors: Submit their site surveys via Airtable form, receive instant PASS/FAIL feedback via email
  • Program Managers: Monitor vendor performance, identify quality issues, track completion rates
  • Executives: View high-level program health, projected savings, and weekly summaries
  • Operations Team: Manage vendor assignments, track Scout site visits, clear backlogs

Key Terms & Metrics
Pass Rate
Percentage of submissions that score ≥95% accuracy against the reference database.
Risk Score
Inverse of pass rate (100 - pass%). Higher = more attention needed. >55 = High Risk.
Scout Visit
Physical site visit by a field technician to gather baseline data before survey.
Reference Database
SQL Server containing the "ground truth" device inventory for each site.
Vendor Assignment
Sites allocated to each vendor for surveying. Tracked for completion monitoring.
Blended Cost
Weighted average cost per site based on survey type mix (Fire, CCTV, Combined).
Agent
Specialized AI assistant (like Code Puppy) that handles specific tasks in the system.
Skill
Reusable automation pattern that agents can invoke for common tasks.

Frequently Asked Questions

❓ Common Questions

  • How often does data refresh? — The pipeline polls Airtable every 30 seconds. Dashboard data updates after each submission is processed.
  • What happens when a submission fails? — The vendor receives an email with the score and a CSV showing which rows didn't match.
  • Can I re-grade a submission? — Yes, change the Processing Status in Airtable back to blank and VUES will re-process it.
  • Why is my pass rate low? — Common causes: wrong site number, missing columns, outdated reference data, or incorrect device naming.
  • How do I add a new vendor? — Vendors are auto-detected from submissions. For Scout assignments, update the Vendor Assignments table in Airtable.
