## Additional Issues Discovered

### Issue A: `_derive_status()` String Mismatch — Airtable Sync Bug
**Severity: HIGH**

The Airtable sync helper checks:
```python
if 'No scout submission' in str(reason):
    return 'Awaiting Scout'
```

But `evaluate_site()` actually writes:
```python
reason_for_decision = "Scout not submitted. Cannot determine survey requirement."
```

`'No scout submission'` is NOT a substring of `"Scout not submitted..."`.
Result: All 90 pending-scout sites will be synced to Airtable with status `"Pending"`
instead of the correct `"Awaiting Scout"`. The Airtable view will misclassify 90 sites.
This does NOT affect the JSON file (correct there), but corrupts the Airtable status field.

**Fix:**
```python
# Change from:
if 'No scout submission' in str(reason):
# Change to:
if 'Scout not submitted' in str(reason):
```
