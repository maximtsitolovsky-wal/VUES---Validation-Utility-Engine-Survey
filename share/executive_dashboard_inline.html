Metric logic

Use formulas you can defend in a meeting.

These are the calculations that matter. No vanity junk, no filler widgets, no fake insight. Just formulas that can survive leadership scrutiny once you wire them into Airtable, Outlook, Power Query, or your scoring workbook.

Pass rate

Count final outcomes only. PASS means pass. Everything else stays out of that numerator.

Pass Rate % = Pass Count / Total Count

Fail rate

Keep actual quality failures separate from broken automation runs.

Fail Rate % = Fail Count / Total Count

Error rate

This tells you whether your system is the problem instead of the vendor.

Error Rate % = Error Count / Total Count

Average fail score

Shows how ugly failed submissions really are, not just how many happened.

Avg Score (Fail) = Average of numeric score where status = FAIL

Average turnaround

Measure from submission creation timestamp to completed scoring timestamp.

Avg Turnaround (s) = Average(Completed Time - Submission Time)

Queue backlog

Immediate operational pain. If this trends upward, you have a choke point.

Queue Backlog = Count(status IN [QUEUED, PROCESSING])
